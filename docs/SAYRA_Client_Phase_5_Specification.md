## SAYRA Enterprise Windows Client: Phase 5 — Admin Integration & Remote Operations
### سند رسمی معماری و خروجی‌های کلیدی پلتفرم (نسخه تجاری - محرمانه)

## ۱. مقدمه و اهداف فاز ۵ (Phase 5 Overview)

فاز ۵ به عنوان **بخش عملیات از راه دور و ادغام سازمانی (Admin Integration & Remote Operations)**، وظیفه اتصال بی‌وقفه، امن و بلادرنگ کلاینت‌های مستقر در ایستگاه‌های کاری (Workstations) را با پنل مدیریت مرکزی (Master Admin Panel) در بستر شبکه محلی (LAN) بر عهده دارد.

این فاز تضمین می‌کند که ادمین مرکز بتواند با کمترین تاخیر (تحت استاندارد زیر ۱۰ میلی‌ثانیه)، کنترل کاملی بر روی جنبه‌های سیستمی، پروسس‌ها، امنیت فیزیکی و منطقی، مانیتورینگ، تبلیغات پویا و خط‌مشی‌های هر کلاینت اعمال کند.

همچنین، این فاز پایداری کلاینت را در سطح **صنعتی و سازمانی (Enterprise grade)** ارتقا داده و مکانیزم‌های دفاع همه‌جانبه، امضاهای رمزنگاری‌شده متقارن و نامتقارن، و تفکیک لایه‌ای سشن‌های ویندوز (Session 0/1 Isolation) را بر روی فرآیندهای مدیریتی پیاده‌سازی می‌کند.

## ۲. خروجی‌ها و دستاوردهای فنی فاز ۵ (Core Technical Deliverables)

این فاز در مجموع شامل **۹ موتور و چارچوب کلیدی** است که به صورت دقیق و ساختاریافته به عنوان خروجی‌های اصلی پیاده‌سازی خواهند شد:

1. **موتور مدیریت از راه دور (Remote Management Engine)**
2. **چارچوب دستورات از راه دور (Remote Command Framework)**
3. **موتور مانیتورینگ زنده (Live Monitoring Engine)**
4. **موتور خط‌مشی و سیاست‌گذاری (Policy Engine)**
5. **موتور تشخیص و عیب‌یابی (Diagnostics Engine)**
6. **موتور تبلیغات هوشمند (Advertisement Engine)**
7. **سیستم مدیریت ناوگان کلاینت‌ها (Fleet Management System)**
8. **سیستم ممیزی و حسابرسی اداری (Administrative Audit System)**
9. **چارچوب عملیات راه دور سازمانی (Enterprise Remote Operations Framework)**

در ادامه، جزئیات معماری، ساختارهای داده، رابط‌های برنامه‌نویسی (Interfaces)، اسکیمای دیتابیس (SQLCipher) و قراردادهای ارتباطی (Protocols) مربوط به هر یک از این خروجی‌ها به صورت کامل تشریح می‌گردد.

### ۳.۱ موتور مدیریت و چارچوب دستورات از راه دور (Remote Management Engine & Command Framework)

این بخش زیرساخت اصلی دریافت، اعتبارسنجی، زمان‌بندی و اجرای دستورات اداری صادر شده از سمت سرور را مدیریت می‌کند.

#### ۳.۱.۱ مولفه‌ها و سرویس‌های فنی (C# Classes & Interfaces)
*   `IRemoteCommandDispatcher`: وظیفه توزیع دستورات به هندلرهای مربوطه را بر عهده دارد.
*   `RemoteCommandEngine`: موتور اصلی پس‌زمینه در کلاینت (Session 0) که صف دستورات و چرخه عمر اجرای آن‌ها را مدیریت می‌کند.
*   `EmergencyCommandExecutor`: وظیفه اجرای آنی و بدون نوبت دستورات اضطراری (نظیر قفل اضطراری کل مرکز یا تخلیه آنی سشن‌ها) را با بالاترین اولویت پردازشی بر عهده دارد.

```csharp
namespace Sayra.Client.Shared.Interfaces
{
    public interface IRemoteCommandDispatcher
    {
        Task<CommandExecutionResult> DispatchAsync(RemoteCommandPayload command, CancellationToken cancellationToken);
    }

    public interface IRemoteCommandEngine
    {
        Task StartEngineAsync(CancellationToken cancellationToken);
        Task QueueCommandAsync(RemoteCommandPayload command);
        Task<CommandStatusDto> GetCommandStatusAsync(Guid commandId);
    }
}
```

#### ۳.۱.۲ لیست دستورات تحت پشتیبانی فاز ۵ (Supported Commands)
*   **مدیریت راه دور سیستمی:** `LOCK_PC` (قفل کردن)، `UNLOCK_PC` (باز کردن کلاینت)، `SHUTDOWN` (خاموش کردن)، `RESTART` (راه‌اندازی مجدد)، `WAKE_ON_LAN` (روشن کردن سیستم‌های خاموش از طریق پکت جادویی کارت شبکه)، `MAINTENANCE_MODE` (حالت تعمیر و نگهداری که سیستم را موقتاً از دسترس کاربران عادی خارج کرده و دسکتاپ ویندوز را برای ادمین باز می‌کند).
*   **کنترل و ناظر پروسس‌ها:** `LAUNCH_GAME` (اجرای خودکار بازی/برنامه خاص)، `CLOSE_GAME` (بستن محترمانه بازی فعال)، `KILL_PROCESS` (خاتمه اجباری پروسس‌های غیرمجاز یا قفل شده)، `RESTART_APPLICATION` (راه‌اندازی مجدد کلاینت WPF)، `RESTART_SERVICE` (راه‌اندازی مجدد سرویس ویندوزی پس‌زمینه)، `RESTART_WORKSTATION` (راه‌اندازی مجدد کل ایستگاه کاری).

#### ۳.۱.۳ اسکیمای پایگاه‌داده تاریخچه دستورات (`CommandHistory` DB Schema)
برای رعایت الزامات حسابرسی و پیگیری وضعیت دستورات در زمان قطعی شبکه، تمام دستورات دریافتی در دیتابیس محلی رمزنگاری‌شده ذخیره می‌شوند.

```sql
CREATE TABLE IF NOT EXISTS RemoteCommandHistory (
    CommandId TEXT PRIMARY KEY NOT NULL,
    Action TEXT NOT NULL,
    TargetPcId TEXT NOT NULL,
    SenderAdminId TEXT NOT NULL,
    PayloadJson TEXT,
    Status TEXT NOT NULL, -- PENDING, EXECUTING, COMPLETED, FAILED
    ErrorMessage TEXT,
    ReceivedAt TEXT NOT NULL,
    ExecutedAt TEXT,
    ExecutionDurationMs INTEGER,
    Signature TEXT NOT NULL -- RSA Signature generated by Master Server
);

CREATE INDEX IF NOT EXISTS IDX_CommandHistory_Status_ReceivedAt
ON RemoteCommandHistory(Status, ReceivedAt);
```

### ۳.۲ موتورهای مانیتورینگ زنده و عیب‌یابی (Live Monitoring Engine & Diagnostics Engine)

این دو موتور با همکاری یکدیگر، پایش لحظه‌ای و ثبت عمیق اطلاعات سخت‌افزاری، نرم‌افزاری و ترافیک شبکه را انجام می‌دهند.

#### ۳.۲.۱ قابلیت‌های مانیتورینگ زنده (Live Status Parameters)
سیستم به صورت غیرمسدودکننده (Non-blocking) پارامترهای زیر را به شکل زمان‌بندی‌شده رصد می‌کند:
*   **Live Performance:** درصد مصرف زمان‌بندی‌شده CPU، رم، حافظه گرافیکی (VRAM) و نرخ فریم بازی در حال اجرا (FPS Tracking).
*   **Live Hardware & Temp:** دمای لحظه‌ای هسته‌های CPU، دمای تراشه گرافیکی GPU و سرعت فن‌ها با استفاده از فراخوانی کتابخانه‌های بومی (بندهای عمیق Win32 API و OpenHardwareMonitor wrappers).
*   **Live Network & Storage:** پهنای باند مصرفی کلاینت، پینگ لوکال به سرور، میزان خواندن/نوشتن هارد دیسک (I/O Read/Write Latency) و فضای خالی پارتیشن‌ها.
*   **Live User & Session:** نام کاربر لاگین شده، سشن فعال، زمان باقیمانده، هزینه سشن جاری و وضعیت قفل کیوسک.

#### ۳.۲.۲ قابلیت‌های عیب‌یابی عمیق (Diagnostics Engine)
با فراخوانی کوئری‌های عمیق سیستم‌عامل، یک مستند "تصویر لحظه‌ای سیستم" تولید می‌کند:
*   **Hardware Diagnostics:** گزارش سلامت هارد دیسک‌ها (S.M.A.R.T)، بررسی خرابی ماژول‌های رم و بررسی دمای بحرانی سخت‌افزار.
*   **Software Inventory:** استخراج لیست دقیق برنامه‌ها و بازی‌های نصب شده روی ویندوز همراه با نسخه‌ها و آدرس نصب آن‌ها.
*   **Process & Driver Inventory:** لیست تمام فرآیندهای در حال اجرا (به همراه هش فایل اجرایی جهت مقابله با چیت‌ها)، درایورهای نصب شده در هسته سیستم‌عامل و سرویس‌های در حال اجرای ویندوز.

#### ۳.۲.۳ سرویس‌ها و اینترفیس‌های فنی
```csharp
namespace Sayra.Client.Shared.Interfaces
{

    public interface IDiagnosticsEngine
    {
        Task<SystemDiagnosticsReport> GenerateFullReportAsync();
        Task<InstalledSoftwareCollection> ScanInstalledSoftwareAsync();
        Task<RunningProcessCollection> CaptureProcessInventoryAsync();
    }
}
```

### ۳.۳ موتور خط‌مشی و سیاست‌گذاری (Policy Engine)

این موتور وظیفه دریافت، اعتبارسنجی و اعمال محدودیت‌ها و قوانین امنیتی و سیستمی را در بستر کلاینت بر عهده دارد. خط‌مشی‌ها به محض همگام‌سازی، بدون نیاز به ریستارت سیستم اعمال (Hot-Applied) می‌شوند.

#### ۳.۳.۱ دسته‌بندی خط‌مشی‌های ویندوز و کلاینت (Policy Categories)
*   **User Policies:** اختصاصی کاربران عادی و ادمین؛ محدود کردن اجرای برخی برنامه‌ها یا مسدود کردن کلیدهای رجیستری ویندوز.
*   **Device & USB Policies:** مسدود کردن اتصال حافظه‌های جانبی غیرمجاز (USB Mass Storage Lockdown) به منظور جلوگیری از انتشار ویروس یا کپی برنامه‌های کلاینت.
*   **Bandwidth & Network Policies:** محدود کردن پهنای باند دانلود و آپلود هر سیستم به صورت لحظه‌ای تا پهنای باند کل گیم‌نت برای بازیکنان بازی‌های آنلاین پایدار بماند.
*   **Time & Session Policies:** اعمال محدودیت‌های زمانی اجرای سیستم‌ها (مثلاً خاموشی خودکار پس از ساعت ۱ بامداد یا خاموشی خودکار پس از خروج کاربر از سشن).
*   **Windows Security Policies:** اعمال فیلتر بر روی رجیستری و گروپ‌پالیسی‌های بومی ویندوز جهت مخفی‌سازی درایوها، بستن کنترل پنل و جلوگیری از تغییر پس‌زمینه ویندوز.

#### ۳.۳.۲ اینترفیس فنی موتور خط‌مشی
```csharp
namespace Sayra.Client.Shared.Interfaces
{
    public interface IPolicyEngine
    {
        Task ApplyPoliciesAsync(PolicyProfile profile);
        Task RemovePoliciesAsync();
        bool IsActionAllowed(PolicyActionType action, object Context);
    }
}
```

#### ۳.۳.۳ اسکیمای دیتابیس قوانین محلی (`LocalPolicies` Database)
```sql
CREATE TABLE IF NOT EXISTS AppliedPolicies (
    PolicyId TEXT PRIMARY KEY NOT NULL,
    Category TEXT NOT NULL, -- USER, DEVICE, NETWORK, GAME, WINDOWS
    RulesJson TEXT NOT NULL,
    VersionCode INTEGER NOT NULL,
    LastUpdatedAt TEXT NOT NULL,
    IsActive INTEGER DEFAULT 1,
    Signature TEXT NOT NULL -- Verified against Server's RSA Public Key
);
```

### ۳.۴ موتور تبلیغات هوشمند (Advertisement Engine)

این موتور برای مراکز بازی بزرگ طراحی شده تا بتوانند از طریق نمایش هدفمند و زمان‌بندی شده بنرها، ویدیوها و صفحات وب تبلیغاتی بر روی مانیتور کلاینت در زمان‌های خاص درآمدزایی کنند.

#### ۳.۴.۱ انواع کمپین‌های تحت پشتیبانی
*   **Image Campaigns:** نمایش پوسترها و بنرهای تبلیغاتی مجلل در پس‌زمینه صفحه قفل سیستم (Lock Screen Backdrop slideshow).
*   **Video Campaigns:** پخش ویدیوهای تبلیغاتی کوتاه در لاندری لانچر یا پیش از شروع سشن کاربری با ابعاد و فشرده‌سازی بسیار سبک.
*   **HTML & Web Campaigns:** رندر فریم‌های پویا با محتوای اینترنتی در لایه‌های رابط کاربری.
*   **Priority & Emergency Broadcasts:** پیام‌های اولویت‌دار مدیریتی یا تبلیغات اسپانسر اصلی که کل صفحه را موقتاً پوشش می‌دهند.

#### ۳.۴.۲ اسکیمای پایگاه‌داده کمپین‌ها (`Campaigns` DB Schema)
```sql
CREATE TABLE IF NOT EXISTS AdCampaigns (
    CampaignId TEXT PRIMARY KEY NOT NULL,
    Name TEXT NOT NULL,
    Type TEXT NOT NULL, -- IMAGE, VIDEO, HTML
    MediaLocalPath TEXT NOT NULL,
    MediaUrl TEXT,
    TargetUrl TEXT,
    Priority INTEGER DEFAULT 1, -- 1 to 10
    DisplayDurationSeconds INTEGER DEFAULT 10,
    StartTime TEXT NOT NULL, -- UTC Start Date
    EndTime TEXT NOT NULL, -- UTC End Date
    DailyActiveHours TEXT, -- JSON Array of hours, e.g., ["14:00-18:00"]
    IsDownloaded INTEGER DEFAULT 0
);

CREATE INDEX IF NOT EXISTS IDX_AdCampaigns_Timeline
ON AdCampaigns(StartTime, EndTime, IsDownloaded);
```

#### ۳.۴.۳ سیستم ثبت گزارشات دیده‌شدن (Ad Telemetry Track)
برای ارائه آمار به کارفرمایان تبلیغاتی، کلاینت تعداد دفعات نمایش هر تبلیغ و کلیک روی آن را ثبت کرده و به صورت رمزنگاری شده برای سرور می‌فرستد.

```sql
CREATE TABLE IF NOT EXISTS AdImpressions (
    ImpressionId TEXT PRIMARY KEY NOT NULL,
    CampaignId TEXT NOT NULL,
    SessionId TEXT,
    ImpressionType TEXT NOT NULL, -- VIEW, CLICK, SKIP
    CreatedAt TEXT NOT NULL,
    IsSynced INTEGER DEFAULT 0
);
```

### ۳.۵ سیستم مدیریت ناوگان کلاینت‌ها (Fleet Management System)

سیستمی پیشرفته در سطح هسته کلاینت که مدیریت دسته‌جمعی سیستم‌ها را به صورت گروه‌های پویا همگام‌سازی می‌کند.

#### ۳.۵.۱ قابلیت‌های بومی مدیریت ناوگان
*   **Machine Groups & Tags:** تخصیص سیستم‌ها به گروه‌های خاص (VIP، معمولی، شبیه‌سازها، تالار استریم) و اعمال همزمان تغییرات و قیمت‌گذاری‌ها به کل گروه.
*   **Dynamic Collections:** تشکیل دسته‌بندی‌های پویا بر اساس مشخصات فیزیکی (مثلاً اعمال خودکار خط‌مشی افزایش اورکلاک بر روی کارت‌های گرافیک RTX 4090 به بالا).
*   **Bulk & Batch Operations:** ارسال دستورات چندگانه همزمان (مثلاً آپدیت کردن و خاموش کردن همزمان تمام سیستم‌های ردیف ۲).
*   **Alert Management:** پردازش بلادرنگ اخطارهای بحرانی سیستم‌ها:
    *   *Critical Alerts:* دمای بیش از ۹۰ درجه کارت گرافیک، جدا شدن کابل شبکه اصلی، توقف پروسس هسته کلاینت.
    *   *Warning Alerts:* پر شدن ۹۵٪ ظرفیت حافظه درایوها، کندی نرخ فریم بازی‌ها به زیر ۴۰ فریم.

### ۳.۶ سیستم حسابرسی و عملیات از راه دور سازمانی (Administrative Audit System & Enterprise Remote Operations Framework)

این بخش وظیفه برقراری لایه امنیتی عمیق، کنترل دسترسی‌های اداری، امضای دیجیتال پکت‌ها و حسابرسی دقیق تمام اقدامات را بر عهده دارد تا هرگونه تلاش برای تقلب یا نفوذ توسط کاربران و ادمین‌های لوکال مرکز مسدود شود.

#### ۳.۶.۱ الزامات و خروجی‌های امنیتی چارچوب
*   **Role-Based Access Control (RBAC):** ادمین‌های لوکال مرکز دارای سطوح دسترسی تفکیک شده هستند (مثلاً ناظر فنی حق خاموش/روشن کردن سیستم‌ها را دارد اما حق تغییر قیمت‌ها یا فعال‌سازی حالت دباگی را ندارد).
*   **Command Authorization & Approval Workflow:** دستورات بسیار حساس سیستمی (مانند بستن اجباری پروسس‌ها یا باز کردن قفل کیوسک به صورت دستی) پیش از اجرا روی کلاینت، امضای دیجیتال مدیر ارشد شبکه را استعلام می‌کنند.
*   **Command Signing via SHA-256 & RSA:** تمام پیام‌های ورودی به نام پایپ و سوکت در کلاینت با گواهی خصوصی سرور تطبیق داده می‌شوند.
*   **Remote Session Logs Download & Remote Config Push:** سیستم انتقال فایل بومی که لاگ‌های عیب‌یابی و پیکربندی‌های جدید را با تکه‌تکه کردن پکت‌ها (Chunked Transport) و اعمال مکانیزم ضد خرابی آپلود/دانلود می‌کند.

#### ۳.۶.۲ دیتابیس حسابرسی اقدامات اداری (`AuditLogs` DB Schema)
در هر ثانیه، هرگونه تغییری توسط ادمین ثبت می‌شود و غیرقابل حذف (Append-Only) است.

```sql
CREATE TABLE IF NOT EXISTS AdminAuditLogs (
    AuditId TEXT PRIMARY KEY NOT NULL,
    CorrelationId TEXT NOT NULL,
    AdminUsername TEXT NOT NULL,
    AdminAction TEXT NOT NULL, -- LOGIN, UNLOCK_OVERRIDE, REGISTRY_SWAP, GAME_KILL
    TargetPcId TEXT NOT NULL,
    Details TEXT,
    Timestamp TEXT NOT NULL,
    SignatureHash TEXT NOT NULL -- Cryptographic chain signature of this and previous record to prevent database editing
);
```

## ۴. لیست فایل‌های خروجی و ساختار فیزیکی فاز ۵ (Physical Assets Checklist)

پس از اتمام توسعه فاز ۵، کلاینت بازی به لیست دقیقی از فایل‌های ساختاریافته مجهز خواهد شد:

### ۴.۱ لایه سورس‌کد هسته کلاینت (`SayraClient` / `Sayra.Client.Shared`)
*   `Sayra.Client.Shared/Models/PolicyModels.cs`: ساختار داده‌ای مربوط به تعاریف خط‌مشی‌ها.

## ۵. ماتریس پایداری و بازیابی در شرایط بحرانی (Stability Matrix)

| سناریوی خطا | تاثیر بر سیستم | فرآیند خودکار بازیابی (Auto-Recovery) |
| :--- | :--- | :--- |
| **قطعی کامل کابل شبکه محلی (LAN Drop)** | قطع ارتباط سوکت مانیتورینگ زنده | کلاینت تلمتری را در دیتابیس `SQLCipher` آفلاین کش می‌کند. سشن کاربر بدون قفل شدن ادامه می‌یابد و تایمر سشن زمان فیزیکی را به صورت آفلاین حفظ می‌نماید. |
| **تلاش گیمر برای بستن فرآیند کلاینت** | غیرفعال شدن موقت مانیتورینگ | موتور ضد تخریب و نگهبان بومی (Watchdog Service) در Session 0 بلافاصله تالار ویندوز را بازآوری کرده، کیبورد را مجدداً قفل کرده و هشدار امنیتی `SECURITY_BREACH` را در صف آفلاین ثبت می‌کند. |
| **خرابی فایل تبلیغات دانلودی** | کرش احتمالی ترنزیشن تصویر | سیستم به جای کرش یا نمایش کادر خالی، بلافاصله تصویر لوگوی دیفالت سایرا را که به صورت Resource تعبیه شده، رندر کرده و فایل خراب را قرنطینه می‌کند. |
| **خطای اعمال رجیستری خط‌مشی‌ها** | باز ماندن بخش‌هایی از ویندوز | سرویس `RegistryWatcher` به محض کوچکترین مغایرت با کانفیگ، دوباره مقدار پالیسی‌ها را بر روی خانه رجیستری مربوطه بازنویسی (Force Overwrite) کرده و گزارش تخلف صادر می‌کند. |

## ۶. نتیجه‌گیری (Summary Verdict)

با اتمام پیاده‌سازی فاز ۵، پلتفرم سایرا از یک کلاینت گیمینگ بومی ساده، به یک **شبکه توزیع‌شده مانیتورینگ و عملیات راه دور در سطح سازمانی (Enterprise Distributed Operations Network)** تبدیل خواهد شد. این لایه یک پارادایم پایدار، سریع و صد درصد امن را جهت اداره هزاران ایستگاه کاری گیمینگ بدون دغدغه‌های امنیتی یا هزینه‌های نگهداری دستی فراهم می‌آورد.
